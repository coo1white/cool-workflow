## Batch — release reviewer exec-capable, high-effort sandbox (Unreleased)

> Cutting v0.1.97 exposed two root causes in the independent release reviewer: when delegated to `builtin:codex`, the reviewer ran `codex exec` in a hardcoded `--sandbox read-only` at `low` reasoning effort. Those defaults are right for a fast delegated worker or a liveness probe, but they make the model structurally unable to RE-RUN the gate (build, tests, dist) its verdict is supposed to be based on — so it fabricated `REJECTED` verdicts (observed twice, each in <65s, far shorter than the ~12-min gate it claimed to have failed). Explicit `CW_CODEX_REASONING_EFFORT`/`CW_CODEX_SANDBOX` still win, and an unknown sandbox fails closed rather than silently downgrading.
